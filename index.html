01 — 비전 및 문제 정의

왜 또 할 일 앱인가?

기존 앱엔 빠진 것이 있다 — "완료의 기쁨을 느끼게 만드는 설계"

🔴 문제

😶
동기부여 부재
할 일을 체크해도 "그래서 뭐가 달라졌지?" 라는 허무함. 완료가 아무런 보상으로 이어지지 않는다.
🗂
빈 목록 진입 장벽
계획 세우기 자체가 일이다. "뭘 어떻게 적어야 하지?"라는 장벽이 앱을 열기도 전에 포기하게 만든다.
📉
꾸준함의 어려움
3일을 잘 쓰다가 하루 빠지면 다시는 안 열게 된다. 연속성을 지키는 메커니즘이 없다.
🎯
큰 목표 ↔ 오늘 할 일 단절
"포트폴리오 완성"이란 목표와 오늘 해야 할 구체적 행동 사이의 연결고리가 없다.

🟢 해결

⚔️
캐릭터 성장 시스템
완료 → XP 획득 → 레벨업 → 외형 변화. 완료의 만족감을 즉각적인 시각적 성장으로 돌려준다.
🤖
AI 자동 계획 생성
오늘 할 일을 대충 써주면 Claude AI가 구조화된 계획표로 변환. 빈 목록 장벽을 제거한다.
🔥
스트릭 시스템
매일 달성하면 스트릭 증가, 마일스톤마다 보상. 유지의 긴장감과 보람을 동시에 제공한다.
🗺
퀘스트 시스템
큰 목표(퀘스트) → 세부 할 일로 분해. 오늘의 행동이 장기 목표 달성에 기여함을 시각화한다.
"단순한 할일 목록 앱이 아닌, 계획 목표 달성 과정에서 재미를 느끼게 하는 어플.
흥미 = 습관 = 계획"

ADR-01 · Flutter 채택 이유

  • ✓ 자체 렌더링 엔진(Impeller)으로 커스텀 애니메이션 안정적
  • ✓ Rive · Lottie 등 게임급 애니메이션 라이브러리 궁합
  • ✓ 단일 코드베이스로 iOS/Android 동시 지원
  • ✓ 동일한 비주얼로 캐릭터 디자인 일관성 보장

기각된 대안

✗ React Native — JS 브릿지 오버헤드, 복잡한 애니메이션 성능 튜닝 부담
✗ 네이티브 (Swift + Kotlin) — 코드베이스 2개, 1인 개발 공수 2배, 일정 내 완성 어려움

데이터 모델

  • 📋 Task — 할 일 (제목, 마감일, XP, 완료여부)
  • 🎯 Goal — 장기 목표 (퀘스트)
  • ⚔️ Character — 레벨, XP, 외형 상태
  • 🔥 Streak — 연속 달성 카운트, 최근 날짜

05 — 리스크 및 대응

위험 요소 관리

식별된 4가지 위험 요소와 사전 대응 전략

R1
캐릭터 애니메이션 구현 복잡도 초과
14주차 UI 작업 지연, 핵심 게임 경험 손상 위험. 대응: 1단계로 Lottie/Rive 무료 에셋으로 대체해 기능 흐름 확보 후 직접 제작 에셋으로 교체.
중간
R2
스트릭 데이터 무결성 오류
자정 경계, 앱 미실행일 처리 오류 시 스트릭 오계산. 대응: 11주차에 스트릭 판정 로직을 별도 모듈로 분리하고 단위 테스트로 경계 케이스 집중 검증.
중간
R3
스코프 크리프 (기능 추가로 일정 초과)
어느 주차에서든 발생 가능. 대응: MoSCoW 문서를 기준으로 삼고 새 아이디어는 Could/Won't로만 분류, 각 주차 시작 시 WBS에서 목표 명시적 확인 후 착수.
높음
R4
앱 성능 저하 (애니메이션 프레임 드롭)
레벨업·완료 애니메이션이 끊기면 핵심 만족감 손상. 대응: Flutter DevTools로 60fps 기준 측정, RepaintBoundary로 렌더링 격리, 저사양 Android 실기기 테스트 필수.
낮음↑중간
06 — AI 플래너 (Should)

Claude API 연동

자동 계획표 생성 + 할 일 추천. Must 완성 후 13주차 구현 예정.

기능 A — 자동 계획표 생성

💬
자유 텍스트 입력
"오전에 수업 듣고, 과제 마무리해야 함..."
🤖
Claude API 호출
JSON 배열로 구조화 출력
📋
임시 목록 표시
사용자 수정 · 확정
⚔️
게임 루프 진입
XP · 캐릭터 반응 적용

기능 B — 할 일 추천

추천 기준

  • 🎯 등록된 장기 목표의 미완료 세부 할 일
  • 📊 최근 7일간 자주 완료한 할 일 패턴
  • ⚠️ 오랫동안 손대지 않은 목표 (방치 감지)

출력 형식

[
  {
    "title": "포트폴리오 작업 30분",
    "reason": "3일째 미진행 중"
  },
  {
    "title": "운동하기",
    "reason": "최근 7일 완료율 85%"
  }
]

주의사항

  • 💰 Claude API 유료 — 무료 티어 내 검증 후 결정
  • 🔒 사용자 데이터 외부 API 전송 UI에서 명시
  • 📴 오프라인 시 직접 입력 모드로 자동 전환
