# ADR-01: 모바일 플랫폼으로 Flutter 채택

- **상태**: 채택됨 (Accepted)
- **날짜**: 2026-05-12

---

## 배경

게임 요소(캐릭터 애니메이션, 레벨업 이펙트, 스트릭 연출)가 핵심인 개인용 플래너 앱을 iOS/Android 양 플랫폼에 출시해야 한다. 플랫폼 선택지는 Flutter, React Native, 네이티브(Swift + Kotlin) 세 가지였다.

## 결정

Flutter 를 사용한다.

## 이유

- 자체 렌더링 엔진(Impeller)으로 복잡한 커스텀 애니메이션을 안정적으로 구현할 수 있다.
- 단일 코드베이스로 iOS/Android를 동시에 지원해 1인 개발 속도에 유리하다.
- `Rive`, `Lottie` 등 게임급 애니메이션 라이브러리와 궁합이 좋다.
- iOS/Android에서 동일한 비주얼을 보장해 캐릭터 디자인 일관성을 유지할 수 있다.

## 기각된 대안

| 대안 | 기각 이유 |
|------|-----------|
| React Native | JS 브릿지 오버헤드로 애니메이션이 많아질수록 성능 튜닝 부담 증가 |
| 네이티브 (Swift + Kotlin) | 코드베이스가 둘로 나뉘어 1인 개발 공수 2배, 일정 내 완성 어려움 |

## 결과

- **긍정**: 애니메이션 품질 확보, 단일 코드베이스 유지
- **트레이드오프**: Dart 언어 학습 필요 (→ `03-risks.md` R1 참고)

## 발표 Q&A 대비

**Q. 왜 React Native 대신 Flutter를 선택했나요?**
> 이 앱의 핵심 경험은 캐릭터 애니메이션과 레벨업 이펙트입니다. React Native는 JS 브릿지 구조상 복잡한 애니메이션에서 성능 튜닝 부담이 커집니다. Flutter는 자체 렌더링 엔진(Impeller)으로 직접 그리기 때문에 게임형 UI에 더 적합하다고 판단했습니다.

**Q. Flutter가 실제 게임 앱에도 쓰이나요?**
> Flutter는 게임 엔진은 아니지만, Flame 엔진을 얹으면 2D 게임 개발도 가능합니다. 이 앱은 게임 '요소'를 가진 생산성 앱이므로 Flutter의 기본 애니메이션 API와 Rive/Lottie 조합으로 충분히 구현 가능합니다. 실제로 여러 게임형 앱이 Flutter로 출시되어 있습니다.

**Q. 나중에 웹이나 데스크톱으로 확장할 수 있나요?**
> Flutter는 동일한 코드베이스로 웹·데스크톱 빌드를 지원합니다. 현재는 모바일에 집중하지만, 향후 확장 시 코드 재작성 없이 대응할 수 있다는 점도 선택 이유 중 하나입니다.
