# WBS (Work Breakdown Structure)

> 3단계 깊이: 1단계(영역) → 2단계(기능 묶음) → 3단계(작업 단위)

---

## 1. 기획 및 설계

### 1.1 요구사항 정의
- 1.1.1 사용자 시나리오 작성
- 1.1.2 MoSCoW 기능 우선순위 정의
- 1.1.3 플랫폼 결정

### 1.2 아키텍처 설계
- 1.2.1 데이터 모델 설계 (Task, Goal, Character, Streak)
- 1.2.2 상태 관리 방식 결정 (Riverpod / Provider)
- 1.2.3 로컬 저장소 전략 결정 (Hive / SQLite / Isar)

### 1.3 UI/UX 설계
- 1.3.1 화면 목록 및 네비게이션 구조 정의
- 1.3.2 와이어프레임 작성 (주요 화면 5종)
- 1.3.3 캐릭터 디자인 방향 결정

---

## 2. 개발 환경 구성

### 2.1 Flutter 프로젝트 초기화
- 2.1.1 Flutter SDK 설치 및 버전 고정
- 2.1.2 프로젝트 생성 및 폴더 구조 세팅
- 2.1.3 핵심 패키지 의존성 추가 (pubspec.yaml)

### 2.2 공통 인프라 구현
- 2.2.1 라우팅 설정
- 2.2.2 테마 / 컬러 시스템 설정
- 2.2.3 로컬 DB 초기화 및 마이그레이션 구조

---

## 3. 핵심 기능 개발 (Must)

### 3.1 할 일 관리
- 3.1.1 할 일 등록 화면 및 입력 폼
- 3.1.2 할 일 목록 조회 (오늘의 할 일 뷰)
- 3.1.3 완료 처리 및 삭제

### 3.2 게임 시스템 — 경험치 & 캐릭터
- 3.2.1 경험치(XP) 계산 로직
- 3.2.2 레벨업 판정 및 이벤트 발행
- 3.2.3 캐릭터 상태 표시 위젯
- 3.2.4 완료 시 캐릭터 반응 애니메이션

### 3.3 게임 시스템 — 스트릭
- 3.3.1 일일 달성 여부 판정 로직
- 3.3.2 스트릭 카운트 저장 및 리셋 처리
- 3.3.3 스트릭 UI 표시 (홈 화면)

---

## 4. 확장 기능 개발 (Should)

### 4.1 장기 목표 (퀘스트) 관리
- 4.1.1 목표 등록 화면 (제목, 기한, 설명)
- 4.1.2 목표 ↔ 세부 할 일 연결
- 4.1.3 목표 진행률 계산 및 표시

### 4.2 보상 시스템
- 4.2.1 스트릭 마일스톤 정의 (7일, 30일 등)
- 4.2.2 마일스톤 달성 시 보상 지급 로직
- 4.2.3 캐릭터 외형 변화 (레벨별 장비·코스튬)

### 4.3 할 일 날짜 관리
- 4.3.1 마감일 지정 UI
- 4.3.2 날짜별 할 일 필터링

---

## 5. UI 완성도

### 5.1 주요 화면 구현
- 5.1.1 홈(데일리 대시보드) 화면
- 5.1.2 할 일 상세 / 편집 화면
- 5.1.3 목표(퀘스트) 목록 화면
- 5.1.4 캐릭터 상태 화면
- 5.1.5 스트릭 히스토리 화면

- 5.2.3 스트릭 달성 축하 연출

### 6.1 테스트
- 6.1.1 핵심 로직 단위 테스트 (XP 계산, 스트릭 판정)
