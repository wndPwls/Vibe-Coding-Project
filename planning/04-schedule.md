# 개발 일정 (10주차 ~ 15주차)

> 12주차: 중간 발표 | 15주차: 최종 발표

---

## 10주차 — 설계 완료 및 환경 구성

**목표**: 개발에 바로 착수할 수 있는 상태 만들기

| 항목 | 내용 |
|------|------|
| 주요 작업 | 데이터 모델 설계, 아키텍처 결정, Flutter 프로젝트 초기화, 와이어프레임 작성 |
| 산출물 | `02-wbs.md`, `03-risks.md`, Flutter 프로젝트 뼈대, 화면 와이어프레임 5종 |
| 검증 방법 | 프로젝트 빌드 성공 확인, 와이어프레임 셀프 리뷰 (시나리오 1~3 커버 여부) |

---

## 11주차 — 핵심 기능 1차 구현

**목표**: Must 기능의 동작하는 프로토타입 완성

| 항목 | 내용 |
|------|------|
| 주요 작업 | 할 일 CRUD, 로컬 DB 연동, XP·레벨 계산 로직, 스트릭 판정 로직 |
| 산출물 | 할 일 등록·완료·삭제 동작, XP/레벨 상태 관리 모듈, 스트릭 카운트 모듈 |
| 검증 방법 | 단위 테스트 통과 (XP 계산, 스트릭 리셋 조건), 로컬 DB 저장·불러오기 수동 확인 |

---

## 12주차 — 중간 발표 ★

**목표**: 핵심 흐름 시연 가능한 데모 완성

| 항목 | 내용 |
|------|------|
| 주요 작업 | 홈 화면 UI 완성, 캐릭터 기본 표시, 완료 체크 애니메이션, 발표 자료 준비 |
| 산출물 | 시나리오 1(할 일 완료 → 캐릭터 반응) 시연 가능한 빌드, 중간 발표 슬라이드 |
| 검증 방법 | 실기기에서 시나리오 1 전체 흐름 시연 성공, 발표 리허설 완료 |

> **중간 발표 범위**: 기획 의도, 핵심 게임 루프(할 일 완료 → XP → 캐릭터 반응), 스트릭 카운트 시연

---

## 13주차 — Should 기능 구현

**목표**: 장기 목표 관리 및 보상 시스템 추가

| 항목 | 내용 |
|------|------|
| 주요 작업 | 장기 목표(퀘스트) CRUD, 목표 ↔ 세부 할 일 연결, 진행률 표시, 마일스톤 보상 로직 |
| 산출물 | 목표 목록 화면, 진행률 위젯, 마일스톤 보상 지급 동작 |
| 검증 방법 | 시나리오 3(목표 설정 → 세부 할 일 완료 → 진행률 반영) 전체 흐름 수동 검증 |

---

## 14주차 — UI 완성도 및 애니메이션 강화

**목표**: 게임 경험의 완성도를 높이고 전체 시나리오 통합 검증

| 항목 | 내용 |
|------|------|
| 주요 작업 | 레벨업 연출, 스트릭 마일스톤 축하 화면, 캐릭터 외형 변화, 전체 화면 연결 및 네비게이션 정리 |
| 산출물 | 5개 주요 화면 완성 빌드, 통합 테스트 결과, TestFlight/내부 테스트 배포본 |
| 검증 방법 | 시나리오 1~3 전체 흐름 실기기 시연 성공, 애니메이션 60fps 유지 확인 |

---

## 15주차 — 최종 발표 ★

**목표**: 완성된 앱 시연 및 회고

| 항목 | 내용 |
|------|------|
| 주요 작업 | 최종 버그 수정, 발표 자료 완성, 시연 시나리오 리허설 |
| 산출물 | 최종 빌드, 최종 발표 슬라이드, 프로젝트 회고 문서 |
| 검증 방법 | 시나리오 1~3 무결점 시연, 발표 리허설 2회 이상 완료 |

> **최종 발표 범위**: 전체 기능 시연, 개발 과정 회고, 미구현 항목(Could/Won't) 및 향후 계획
