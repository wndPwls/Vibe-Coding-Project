# 기능 스펙: AI 플래너 (자동 계획표 생성 + 할 일 추천)

이 파일은 두 가지 AI 기능을 함께 다룬다.

| 기능 | 트리거 | 핵심 동작 |
|------|--------|-----------|
| **자동 계획표 생성** | 사용자가 오늘 일정을 자유 텍스트로 입력 | 입력을 파싱해 구조화된 할 일 목록으로 변환 |
| **할 일 추천** | 앱이 자동 분석 (목표·완료 패턴 기반) | 오늘 하면 좋을 할 일을 선제적으로 제안 |

---

## 기능 A — 자동 계획표 생성

### 한 줄 설명

오늘 할 일을 대충 텍스트로 입력하면, AI가 자동으로 구조화된 계획표(할 일 목록 + 순서 + 시간 배분)를 만들어준다.

## 해결하는 문제

계획 자체를 세우는 것이 어려운 사람들은 빈 할 일 목록 앞에서 막힌다. "뭘 어떻게 적어야 하지?"라는 장벽이 앱 진입을 방해한다. 이 기능은 그 장벽을 없앤다.

---

## 사용자 시나리오

**상황**: 사용자가 오늘 해야 할 일은 대충 알지만, 구체적인 계획을 세우기 귀찮다.

1. 홈 화면에서 "오늘 뭐 해야 해?" 입력창을 연다.
2. 자유롭게 타이핑한다.
   > 예시 입력: *"오전에 수업 듣고, 점심 먹고 과제 마무리해야 함. 저녁엔 운동도 가고 싶고, 자기 전에 책 좀 읽기"*
3. AI가 이를 분석해 오늘의 할 일 목록을 자동 생성한다.
   > 예시 출력:
   > - [ ] 수업 듣기
   > - [ ] 점심 식사
   > - [ ] 과제 마무리
   > - [ ] 운동
   > - [ ] 독서 (취침 전)
4. 사용자가 목록을 확인하고 수정·추가·삭제한 뒤 "오늘 시작!" 버튼으로 확정한다.
5. 확정된 항목이 오늘의 할 일로 등록되고, 게임 루프(XP·캐릭터 반응)가 적용된다.

---

## 기술 구현 방향

### LLM 연동
- **Claude API** (Anthropic) 사용 권장 — 한국어 자연어 이해 품질 우수
- 입력 텍스트를 프롬프트로 변환해 API 호출, 구조화된 JSON 응답 파싱

### 프롬프트 설계 방향
- 역할: "당신은 하루 계획을 도와주는 어시스턴트입니다."
- 출력 형식: JSON 배열 (`[{ "title": "...", "order": 1 }, ...]`)
- 제약: 할 일은 최대 8개, 각 제목은 15자 이내로 간결하게

### 데이터 흐름
```
사용자 자유 입력
    → Claude API 호출 (프롬프트 + 입력 텍스트)
    → JSON 응답 파싱
    → 임시 할 일 목록 표시 (수정 가능)
    → 사용자 확정
    → 로컬 DB 저장 + 게임 루프 진입
```

### 오프라인 대응
- API 호출 실패 시 "직접 입력 모드"로 자동 전환, 사용자에게 안내 메시지 표시

---

## 게임 연동

- AI가 생성한 할 일도 일반 할 일과 동일하게 XP·캐릭터 반응 적용
- "AI 계획으로 하루 완주" 달성 시 별도 보상 고려 (Could 항목)

## 기능 B — AI 기반 할 일 추천

### 한 줄 설명

사용자가 아무것도 입력하지 않아도, 앱이 목표·완료 패턴을 분석해 "오늘 이런 거 해보는 건 어때요?" 를 먼저 제안한다.

### 사용자 시나리오

**상황**: 사용자가 앱을 열었지만 오늘 뭘 해야 할지 막막하다.

1. 홈 화면에 "오늘의 추천" 카드가 표시된다.
2. 카드에는 2~3개의 추천 할 일이 나열된다.
   > 예시: *"포트폴리오 작업 30분", "운동하기", "어제 못 끝낸 과제 마무리"*
3. 사용자가 원하는 항목을 탭하면 오늘의 할 일로 바로 추가된다.
4. 추천을 모두 무시해도 괜찮다 — 강제성 없음.

### 추천 기준 (분석 데이터)

- 등록된 장기 목표와 아직 완료 안 된 세부 할 일
- 최근 7일간 자주 완료한 할 일 유형 (패턴)
- 오랫동안 손대지 않은 목표 (방치 감지)

### 기술 구현 방향

- 로컬 데이터(목표, 완료 기록)를 요약해 Claude API 프롬프트로 전달
- 출력 형식: JSON 배열 (`[{ "title": "...", "reason": "..." }, ...]`)
- `reason` 필드를 UI에서 짧게 노출해 추천 이유를 투명하게 표시
- 추천 주기: 하루 1회 (앱 첫 실행 시 갱신)
